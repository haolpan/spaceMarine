---

## Summary

Add faction-accurate multi-character silhouettes for all entities (US1) and a three-tier Bolter
upgrade system with collectible power-up tokens (US2). Implemented additively across the two-file
architecture: game.py gains `WeaponTier`, `PowerUpToken`, `dx`-extended `BolterRound`, and token
lifecycle logic; space_marine.py gains multi-row art rendering and HUD tier label. All 32 existing
tests continue to pass; new tests cover token spawn, collection, tier advance, tier reset, and
diagonal round movement.

**Rollout order**: US2 logic (game.py + tests) → US2 renderer (HUD label + token `*` symbol) →
US1 renderer (silhouettes + boss art). This keeps the logic layer fully tested before any visual work.

#### Step 1.2 — Add BolterRound.dx field

Add `dx: int = 0` to `BolterRound`. All existing round instantiations use positional/keyword args
`BolterRound(x=..., y=...)` and are unaffected.

Update `update()` round movement loop:
```python
for r in self.rounds:
    r.x += r.dx   # NEW: apply drift before y movement
    r.y -= 1
```

Collision check uses `r.x` (already post-drift after this change).

#### Step 1.3 — Modify handle_input() firing logic

Replace single-round spawn with tier-aware multi-round spawn:

```python
elif key == ord(' '):
    px = self.player.x
    tier = self.player.weapon_tier
    if tier == WeaponTier.STANDARD:
        self.rounds.append(BolterRound(x=px, y=PLAYER_ROW - 1))
    elif tier == WeaponTier.TWIN_LINKED:
        self.rounds.append(BolterRound(x=px - 1, y=PLAYER_ROW - 1))
        self.rounds.append(BolterRound(x=px + 1, y=PLAYER_ROW - 1))
    elif tier == WeaponTier.STORM_BOLTER:
        self.rounds.append(BolterRound(x=px,     y=PLAYER_ROW - 1, dx=0))
        self.rounds.append(BolterRound(x=px - 1, y=PLAYER_ROW - 1, dx=-1))
        self.rounds.append(BolterRound(x=px + 1, y=PLAYER_ROW - 1, dx=+1))
```

#### Step 1.4 — Add PowerUpToken dataclass and constants

Add after `BossProjectile` dataclass. Add `POWERUP_DROP_CHANCE` and `POWERUP_FALL_SPEED` constants.

#### Step 1.5 — Add GameState.tokens and _update_tokens()

Add `self.tokens: List[PowerUpToken] = []` to `__init__`.
Add `self.tokens = []` to `_spawn_wave()` and `reset()`.

Add `_update_tokens()` method:

```python
def _update_tokens(self) -> None:
    surviving = []
    for token in self.tokens:
        token.y += POWERUP_FALL_SPEED
        if int(token.y) == PLAYER_ROW and token.x == self.player.x:
            # Collect
            if self.player.weapon_tier == WeaponTier.STANDARD:
                self.player.weapon_tier = WeaponTier.TWIN_LINKED
            elif self.player.weapon_tier == WeaponTier.TWIN_LINKED:
                self.player.weapon_tier = WeaponTier.STORM_BOLTER
            # STORM_BOLTER: no-op (FR-013)
        elif int(token.y) <= PLAYER_ROW:
            surviving.append(token)
        # int(token.y) > PLAYER_ROW → dropped, removed (FR-020)
    self.tokens = surviving
```

Call `self._update_tokens()` from `update()` after `_update_boss_projectiles()`.

#### Step 1.6 — Spawn token on enemy kill (_check_collisions)

After `self.player.score += e.score_value` and before `dead_enemy_indices.add(ei)`:

```python
if random.random() < POWERUP_DROP_CHANCE:
    self.tokens.append(PowerUpToken(x=e.x, y=float(int(e.y))))
```

Also add token spawn for boss kill (after boss score add).

#### Step 1.7 — Tier reset on life loss

Every location where `self.player.lives` is decremented, add tier reset:

1. `_descend_enemies()`: after `self.player.lives = max(0, self.player.lives - 1)`
2. `_update_boss_projectiles()`: after life deduction
3. Any future life-loss sites

```python

```

### Phase 4: US1 Renderer — Player Silhouette (space_marine.py)

#### Step 4.1 — Evaluate row 22 player art feasibility

Row 22 is the flavor text + controls bar. This row cannot host player body art without overwriting
the flavor text. The player renders at PLAYER_ROW (row 23) only.

**Resolution**: Player symbol `[℧]` at row 23 spans 3 columns and displays the Ultima chapter mark.
FR-001 ("minimum 2 rows and 3 columns") is met by width (3 cols) at the play area boundary.
FR-002 (Ultima mark) is met by `℧` (U+2127).

If the design team requires a true second row, the flavor/controls bar would need to move to the
HUD area — this is out of scope for this feature. Document this trade-off in the task.

**No code change required for player art** unless the team decides to restructure the status bar.

## Edge Cases

| Case | Implementation |
|---|---|
| Multiple tokens falling simultaneously | `gs.tokens: List[PowerUpToken]` — each iterated independently in `_update_tokens()` |
| Token and BolterRound same cell | No interaction — token update is separate from collision loop (FR-019) |
| Token falls past player row | Removed when `int(token.y) > PLAYER_ROW` (FR-020) |
| Player dies while token mid-fall | `_spawn_wave()` called from `reset()` clears `gs.tokens = []` |
| Storm diagonal round at boundary | Existing `r.y >= PLAY_AREA_TOP` filter removes round; no x-boundary check needed |
| Tier reset + active rounds | Rounds already in flight continue with original dx — reset only affects next fire |
| Token spawned at boss kill | Same 15% check in boss collision handler |
